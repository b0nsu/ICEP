## 5. 데이터 파일

### 5.1 문법 규칙 (형식)

검사 목표:
- 각 데이터 파일의 필드 개수, 레코드 접두어, ID 형식, 날짜/시각 형식, 정수 형식을 확인한다.
- 빈 줄과 `#` 주석 줄이 무시되는지 확인한다.

수행한 주요 테스트:
- `users.txt`, `rooms.txt`, `reservations.txt` 필드 수 오류
- 잘못된 `userId`, `roomId`, `reservationId`
- 잘못된 `loginId`, `userName`, 짧은 `password`
- 잘못된 `roomStatus`, `reservation status`
- 잘못된 `date`, `startTime`, `endTime`, `createdAt`, `checkedInAt`, `NOW`
- 잘못된 `maxCapacity`
- 잘못된 reservation `userId`, `roomId`
- `password`의 개행 표기 문자열(`\\n`)에 대한 추가 방어
- 주석 줄 / 빈 줄 무시

대표 예상 결과:
- 형식이 잘못된 레코드는 `[파일 오류] 파일명 n행: 원인` 형식으로 출력하고 프로그램 시작을 중단해야 한다.
- 빈 줄과 `#`로 시작하는 줄은 무시하고 나머지 정상 레코드만 읽어야 한다.

### 5.2 의미 규칙 (추가 조건)

검사 목표:
- 데이터 전체 수준에서 유일성, 참조 무결성, 상태-필드 조합, 정원 조건, 최소 admin 존재 여부를 확인한다.

수행한 주요 테스트:
- 중복 `userId`, `loginId`, `roomId`, `reservationId`
- `role=admin` 계정이 하나도 없는 경우
- 존재하지 않는 `userId`, `roomId` 참조
- `CHECKED_IN/COMPLETED`인데 `checkedInAt=-`
- `RESERVED/NO_SHOW`인데 `checkedInAt` 값 존재
- `partySize > room.maxCapacity`
- `partySize=0`
- `startTime >= endTime`
- 예약 길이 5시간

대표 예상 결과:
- 의미 규칙을 위반한 데이터는 정상 데이터로 간주하지 않고 시작 단계에서 즉시 중단해야 한다.

### 5.3 부가 확인 항목

검사 목표:
- 예약 충돌 확인과 영향 예약 판단의 공통 조건이 기능별로 일관되게 적용되는지 확인한다.

#### 5.3.1 예약 충돌 확인

수행한 주요 테스트:
- 데이터파일 룸 충돌
- 데이터파일 사용자 충돌
- 예약 가능 조회 사용자 충돌 거절
- 예약 신청 룸 충돌 / 사용자 충돌 거절
- 예약 신청 같은 룸 / 같은 회원 인접 시간 허용
- 예약 조정 대상 룸 시간 충돌 거절

대표 예상 결과:
- 같은 룸 또는 같은 회원의 `RESERVED/CHECKED_IN` 예약이 시간 구간 기준으로 겹치면 허용되면 안 된다.
- `[start, end)` 반열린 구간 기준으로 종료 시각과 시작 시각이 맞닿는 경우는 충돌로 보지 않아야 한다.

#### 5.3.2 영향 예약

수행한 주요 테스트:
- 체크인 창 `RESERVED` 정원 변경 영향 처리
- 체크인 창 `RESERVED` 휴업 영향 처리
- 영향 예약 이동 성공
- 영향 예약 처리 중 `2) 해당 예약 취소` 선택 성공
- 영향 예약 전체 취소 복구

대표 예상 결과:
- 정원 변경 또는 임시 휴업 시 영향 예약이 있으면 단순 반영이 아니라 영향 예약 처리 절차로 넘어가야 한다.

### 5.4 무결성 확인 및 처리

검사 목표:
- 데이터 파일 검증 실패 시 시작 단계에서 중단되는지 확인한다.
- 현재 시각 변경 후 상태 자동 갱신과 저장이 정상 반영되는지 확인한다.

수행한 주요 테스트:
- 잘못된 파일 문법 / 의미 규칙 위반 시 시작 중단
- 현재 시각 변경 후 `RESERVED -> NO_SHOW`, `CHECKED_IN -> COMPLETED`

대표 예상 결과:
- 시작 단계 오류가 있으면 조용히 무시하지 않고 메시지 출력 후 중단해야 한다.

## 7. 기타 처리

### 7.1 상태 자동 갱신

검사 목표:
- 프로그램 시작 및 현재 시각 변경 이후 `NO_SHOW`, `COMPLETED` 자동 전환이 정상적으로 반영되는지 확인한다.

수행한 주요 테스트:
- 프로그램 시작 시 `RESERVED -> NO_SHOW`
- 프로그램 시작 시 `CHECKED_IN -> COMPLETED`
- 현재 시각 변경 시 `RESERVED -> NO_SHOW`, `CHECKED_IN -> COMPLETED`
- 체크인 창 안의 `RESERVED` 유지
- 종료 시각 도달 시 `COMPLETED`

대표 예상 결과:
- 종료 상태는 다시 되돌리지 않고, 전이 대상만 정확히 갱신해야 한다.

### 7.2 필수 파일 누락 및 시작 단계 오류 처리

검사 목표:
- `data/` 폴더 및 필수 파일 누락 시 자동 생성과 시작 단계 오류 중단 규칙을 확인한다.

수행한 주요 테스트:
- 필수 파일 자동 생성
- 잘못된 파일 문법 / 의미 규칙 위반 시 시작 중단

대표 예상 결과:
- 자동 생성 가능하면 생성 후 계속 진행하고, 불가능하거나 검증 실패면 `[파일 오류]`와 함께 중단해야 한다.

### 7.3 오류 메시지와 메뉴 복귀

검사 목표:
- 메뉴 번호 오류는 같은 단계 재입력, 일반 입력 형식 오류는 상위 메뉴 복귀라는 기획서 규칙을 확인한다.

수행한 주요 테스트:
- 비로그인 / member / admin 메뉴 번호 오류
- 룸 컨디션 관리 하위 입력 오류 후 룸 컨디션 관리 메뉴 복귀
- 회원가입 / 예약 / 체크인 입력 오류 후 상위 메뉴 복귀

대표 예상 결과:
- 메뉴 번호 오류는 같은 메뉴 단계에서 다시 입력받아야 하고, 그 외 기능 입력 오류는 직전 메뉴를 다시 보여 주어야 한다.

### 7.4 파일 직접 수정 시 처리

검사 목표:
- 사용자가 프로그램 종료 후 데이터 파일을 직접 수정했을 때 문법/의미 규칙 위반이 다음 실행 시 탐지되는지 확인한다.

수행한 주요 테스트:
