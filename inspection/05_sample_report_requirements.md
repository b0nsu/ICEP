## 핵심 기준

- 검사 종류는 `최종 완제품에 대한 전체 통합 검사`만 보고한다.
- TC 기준은 `기획서`다. 단위 검사나 중간 단계 통합검사는 보고서 본문 기준에서 제외한다.
- 각 TC는 최소한 `ID / 목표 / 입력 / 방법 / 예상 결과`를 가져야 한다.
- 각 TC가 `기획서의 어떤 절/항`을 확인하는지 독자가 바로 알 수 있어야 한다.
- 같은 목표를 확인하는 여러 TC는 `한 표나 한 묶음`으로 정리하는 것이 권장된다.
- 예상 결과는 프로그램 실행 결과를 보고 거꾸로 쓰지 않고 `기획서 기준`으로 적어야 한다.
- CLI 프로그램은 스크린샷 대신 `터미널 텍스트 복붙`을 써도 된다.
- 같은 목표에서 입력만 다른 경우 `대표 로그 1개 + 표` 방식이 허용된다.
- 목차와 페이지 번호가 필요하다.

## 첫 페이지에 보여야 하는 항목

- 팀명: `C07`
- 문자열: `1차 검사 보고서`
- 프로젝트 주제명

## 본 저장소 데이터에 대한 적용 방침

- 기존 `inspection/evidence/TC-*`는 raw evidence 저장소로 유지한다.
- 보고서 본문용 표는 `기획서 흐름`에 맞춰 별도 묶음형 문서로 재배치한다.
- 대부분 PASS 케이스는 실제 결과를 간단히 쓰거나 `상동`으로 줄일 수 있지만, 현재는 복붙 편의를 위해 핵심 결과를 한 줄씩 남긴다.
- 파일 저장/삭제/상태 변경처럼 화면에 잘 안 보이는 결과는 `before/after` 스냅샷 경로를 함께 적는다.
