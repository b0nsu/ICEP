## 추가 반영된 세부 테스트

최신 회귀에서는 위 대표 시나리오 외에도 아래 세부 분기/경계/데이터파일 검사 케이스를 추가로 수행하며, 현재 모두 통과했다.

### 메뉴/세션 흐름

- 비로그인 메뉴 `0` 종료
- member 로그아웃 후 비로그인 메뉴 복귀
- admin 로그아웃 후 비로그인 메뉴 복귀

### 현재 시각 변경

- 잘못된 날짜/시각 값 입력 시 형식 오류 처리

### 예약번호 존재 여부 분기

- 존재하지 않는 예약번호로 예약 취소 시 오류
- 존재하지 않는 예약번호로 체크인 시 오류
- 존재하지 않는 예약번호로 예약 조정 시 오류

### 예약 신청 입력 구분

- 잘못된 날짜 형식 입력 시 오류
- 존재하지 않는 룸 ID 입력 시 오류
- 룸 ID 형식 오류 입력 시 오류
- 위 세 케이스가 서로 다른 분기로 처리되는 것 확인

### 데이터 파일 검사 케이스

- `users.txt` 필드 수 오류
- `rooms.txt` 필드 수 오류
- `rooms.txt`의 잘못된 `roomId`
- `rooms.txt`의 빈 `roomName`
- `reservations.txt`의 잘못된 `reservationId`
- `reservations.txt`의 존재하지 않는 `userId` 참조
- `reservations.txt`의 존재하지 않는 `roomId` 참조
- `CHECKED_IN/COMPLETED` 상태인데 `checkedInAt = -`
- `RESERVED/NO_SHOW` 상태인데 `checkedInAt` 값 존재
- `partySize > room.maxCapacity`
- 같은 룸 활성 예약 충돌
- 같은 사용자 활성 예약 충돌

### 추가 경계값/권한 분기

- 회원가입 `loginId` 최대 길이 초과
- 회원가입 `userName` 최소 길이 미달
- 타인 예약 취소 시도
- 타인 예약 체크인 시도
- 예약 조정 대상 룸 정원 부족
- 예약 조정 대상 룸 시간 충돌
