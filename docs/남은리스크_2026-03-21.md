# 남은 리스크 메모

기준일: 2026-03-21

본 문서는 현재 스터디룸 예약 CLI 코드베이스에서 아직 남아 있는 운영 리스크를 기록한 메모다.

## 1. 진행 중 RESERVED 예약의 룸 휴업 정책 미확정

- 현재 구현은 `CHECKED_IN` 예약이 있는 룸의 즉시 휴업은 막는다.
- 반면 이미 시작됐지만 아직 체크인하지 않은 `RESERVED` 예약이 있는 룸은 미래 예약으로 보지 않아 영향 예약 처리 대상에 포함되지 않을 수 있다.
- 그 경우 룸을 `CLOSED`로 바꾼 뒤 해당 사용자의 체크인이 거부될 수 있다.
- 이 항목은 현재 기획 문서의 정책이 모호해 코드상 허용/금지 기준을 추가 확정할 필요가 있다.

## 2. 저장 원자성 미보장

- 저장은 `users.txt`, `rooms.txt`, `reservations.txt`, `system_time.txt`를 순차적으로 직접 덮어쓰는 방식이다.
- 저장 중간에 I/O 오류나 비정상 종료가 발생하면 일부 파일만 갱신된 상태가 남을 수 있다.
- 이후 재실행 시 파일 간 상태 불일치로 인해 무결성 오류가 발생할 가능성이 있다.
- 향후에는 임시 파일 생성 후 검증하고 원본을 교체하는 방식으로 보강하는 것이 안전하다.

## 3. 장애 주입 테스트 미구현

- 현재 자동 테스트는 주요 기능 흐름과 경계값을 실제 CLI 실행 기준으로 폭넓게 검증한다.
- 다만 파일 권한 오류, 저장 중 실패, 비정상 종료 같은 OS/파일시스템 수준 장애는 자동화하지 않았다.
- 따라서 기능 정확성은 높아졌지만 장애 내성은 별도 테스트가 더 필요하다.
