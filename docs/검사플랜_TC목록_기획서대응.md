# 검사 플랜 TC 목록 - 기획서 대응 기준

## 작성 원칙

- 분류는 운영 편의를 위해 `시간 / 데이터 / 어드민 / 유저 / 영향예약 / 룸컨디션` 6개로 나눈다.
- TC 번호와 기준은 반드시 기획서 절 번호를 따른다.
- `7.3 오류 메시지와 메뉴 복귀`는 각 기능의 실패 TC에 흡수하여 확인한다.
- 아래 입력 예시는 현재 기본 데이터셋을 기준으로 한다.
- 2차 최종안 기준에서는 모든 `RESV` 입력/기대값을 `extensionCount` 포함 12필드로 작성한다.
- 최근 7일 판정은 `now - 7days < eventTime <= now` 반열린 구간으로 검사한다.
- 패널티 해제 시각은 마지막 노쇼 발생 시각 + 7일로 검사한다.

## 현재 기본 데이터셋 기준

- 현재 시각: `2026-03-20 09:00`
- admin: `user001 / admin1234`
- member: `user101 / 12345`, `user102 / 1234`
- room: `R101`, `R102`, `R103` 모두 `OPEN`
- 예약 파일: 기본적으로 비어 있는 상태에서 시작

## 사용 팁

- 보고서 목차는 위 6개 분류가 아니라, 기획서 절 번호 순서대로 다시 풀어 쓰는 것이 안전하다.
- 팀 내부 분업은 위 6개 묶음으로 해도 되지만, 보고서에는 `기획서 대응` 열을 반드시 넣는다.
- 각 TC는 가능하면 독립적으로 실행하고, 예약번호는 매번 초기화된 데이터 기준으로 다시 기록한다.
