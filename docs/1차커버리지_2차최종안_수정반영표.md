# 1차 커버리지의 2차 최종안 수정 반영표

## 목적

이 문서는 1차 검사 커버리지에서 2차 최종 제출안 때문에 수정되어야 하는 항목을 정리한 반영표이다.
1차의 검사 축은 유지하되, 예약 데이터 형식과 2차 신규 정책 때문에 입력 데이터, 기대 결과, 추가 TC가 필요한 항목을 갱신한다.

## 전체 반영 상태

- 기존 1차 회귀 테스트 축 유지
- 자동 테스트 수: `169`개 기준에서 `198`개 기준으로 확대
- 2차 최종안 추가/수정 커버리지: `TC170`~`TC198`
- 실행 기준: `cmd /c gradlew.bat clean check --no-daemon`
- 최종 결과: `198/198 PASS`

## 1차 커버리지 수정 대상

| 1차 커버리지 영역 | 기존 기준 | 2차 최종안 기준 | 반영 상태 |
| --- | --- | --- | --- |
| reservations.txt 문법 | RESV 11필드 | RESV 12필드, 마지막 `extensionCount` 필수 | `TC172`, `TC193` 반영 |
| RESV 기대 레코드 | `...|checkedInAt` 또는 `...|-`로 종료 | `...|checkedInAt|extensionCount`, 기본값 `0` | 기존 회귀 테스트 기대값 이관 |
| 예약 데이터 의미 검사 | 1~4시간 예약 중심 | 비연장 예약은 4시간 이하, 연장 예약은 최대 5시간 | `TC190` 및 데이터 검증 반영 |
| extensionCount 검증 | 없음 | 정수, 0 이상, 3 이하 | `TC198` 반영 |
| 상태별 extensionCount | 없음 | RESERVED/NO_SHOW는 0, CHECKED_IN/COMPLETED는 1 이상 가능 | 데이터 검증 반영 |
| ID 자동 생성 | 다음 번호 생성 | `user999`, `rv9999` 상한 도달 시 생성 거부 | `TC170`, `TC171` 반영 |
| ID 하한 | 형식 중심 | `user000`, `rv0000` 데이터 거부 | `TC194`, `TC195` 반영 |
| 로그인 출력 | 로그인 성공/실패 중심 | 회원 상태, 패널티 상태, 패널티 해제 시각 출력 | `TC173`, `TC175`, `TC176` 반영 |
| 예약 신청 | 예약 가능 조건 중심 | 패널티 대상은 새 예약 신청 거부 | `TC173`, `TC174` 반영 |
| 최근 7일 판정 | 1차에는 없음 | `now - 7days < eventTime <= now` 반열린 구간 | `TC175`~`TC178` 반영 |
| 패널티 계산 | 1차에는 없음 | 마지막 노쇼 발생 시각 + 7일 | `TC173`, `TC174`, `TC176` 반영 |
| 우수회원 판정 | 1차에는 없음 | 최근 7일 완료 2건 이상 + 같은 구간 노쇼 없음 | `TC177`~`TC182` 반영 |
| member 메뉴 | 1~6, 0 | `7. 예약 연장(우수회원 전용)` 추가 | `TC179`~`TC191` 반영 |
| 예약 연장 | 없음 | 우수회원 선검사 후 예약번호 입력, 1시간 단위 연장 | `TC179`, `TC180`, `TC183` 반영 |
| 연장 경계 | 없음 | 종료 30분 전 포함, 종료 이전까지 가능 | `TC187`, `TC188` 반영 |
| 연장 제한 | 없음 | 최대 3회, 총 5시간 이하, 날짜 넘김 금지 | `TC189`~`TC191` 반영 |
| 연장 충돌 | 없음 | 같은 룸/같은 회원 활성 예약과 충돌 금지 | `TC184`, `TC185` 반영 |
| 룸 상태 | 예약/체크인 중심 | CLOSED 룸에서는 연장도 거부 | `TC186` 반영 |
| 자동 상태 갱신 | CHECKED_IN 종료 시 완료 | 연장된 endTime 기준으로 완료 | `TC192` 반영 |
| createdAt 의미 검사 | 형식 중심 | `createdAt <= startDateTime` | `TC196` 반영 |
| checkedInAt 의미 검사 | 상태별 존재 여부 중심 | 체크인 가능 구간 안이어야 함 | `TC197` 반영 |
| 조회 출력 | 예약 기본 열 | 연장 횟수 열 추가 | 기존 조회 TC 기대값 이관 |
| 관리자 이동/영향 처리 | RESV 전후 출력 | 12필드 RESV와 extensionCount 보존 | 기존 관리자 TC 기대값 이관 |

## 수정하지 않아도 되는 1차 커버리지 축

다음 영역은 2차 최종안에서도 기본 검사 축을 그대로 유지한다.

- 회원가입 입력 형식 검증
- 로그인 성공/실패
- 현재 가상 시각 변경과 과거 시각 거부
- 예약 가능 룸 조회의 기본 조건
- 예약 신청의 기존 1~4시간 제한
- 예약 취소
- 체크인 가능 구간과 상태 전이 기본 규칙
- 관리자 전체 예약 조회, 예약 조정, 룸 컨디션 관리의 기본 흐름
- 파일 누락 자동 생성과 시작 단계 오류 처리

단, 위 기능에서 RESV 레코드를 입력하거나 기대 출력으로 비교하는 경우에는 모두 12필드 기준으로 수정한다.
