# 구현 메모 (추가 점검)

## 저장 원자성
- 현재 구현(`src/TextDataStore.java`)은 `Files.writeString`으로 대상 파일에 직접 저장한다.
- 임시 파일 생성 후 재검증/원본 교체 방식은 미구현이다.
- 따라서 저장 중 비정상 종료가 발생하면 일부 파일만 반영될 수 있는 리스크가 있다.

## 오류 상황 재실행 확인

### 1) 필수 파일 권한 오류
- 재현: `users.txt`를 쓰기 불가로 설정 후 실행
- 관찰 출력:
  - `[파일 오류] users.txt: 파일 쓰기 권한이 없습니다.`
  - `[안내] 프로그램을 종료합니다.`
- 동작: 메인 메뉴 진입 전 즉시 종료

### 2) 손상된 파일 문법 오류
- 재현: `reservations.txt`에 필드 개수 불일치 라인 주입 후 실행
- 관찰 출력:
  - `[파일 오류] reservations.txt 1행: 필드 개수가 올바르지 않습니다.`
  - `[안내] 프로그램을 종료합니다.`
- 동작: 메인 메뉴 진입 전 즉시 종료

### 3) 저장 실패(I/O)
- 재현: 실행 중 예약 생성 직전에 `reservations.txt`를 쓰기 불가로 전환
- 관찰 출력:
  - `[파일 오류] reservations.txt: 파일 쓰기 중 오류가 발생했습니다: .../reservations.txt`
  - `[안내] 프로그램을 종료합니다.`
- 동작: 데이터 수정 시도 단계에서 즉시 종료
