# 테스트 체크리스트: 룸 컨디션 관리 프로세스

**문서명**: 테스트 체크리스트 - 룸 컨디션 관리 프로세스  
**작성일**: 2026-04-23 (수정: 2026-04-23)  
**대상**: 관리자의 룸 정원 및 상태 변경 기능  
**범위**: 그림 6(룸 컨디션 관리) + 그림 7(영향예약 처리) 흐름도

## 개요

이 문서는 **그림 6: 룸 컨디션 관리 프로세스** 흐름도의 모든 경로를 테스트하기 위한 체크리스트를 제공한다.

포함 범위:
1. **그림 6: 룸 컨디션 관리 프로세스** (Path 1~2, Path 5)
   - 현재/미래 **예약 존재 여부**
   - **정원 미달 예약 존재 여부**
   - **룸 상태 변경 (OPEN/CLOSED)**
2. **그림 7: 영향예약 처리 흐름도** (Path 3~4)
   - 정원 변경 시 영향 예약 처리
   - 임시 휴업 시 영향 예약 처리

## Path 3: 영향예약 처리 (정원 변경 시 예약 인원 초과)

**트리거 조건**: 기존 예약의 partySize보다 **작은 정원으로 변경** → 영향예약 흐름도 진입

**영향예약 선택지 정의**:
- `선택 1 (이동)`: 해당 룸에 걸린 예약을 다른 방으로 이동시켜 영향예약 상황을 해결
- `선택 2 (해당 예약 취소)`: 해당 룸에 걸린 예약 자체를 취소하여 영향예약 상황을 해결
- `선택 0 (전체 취소)`: 영향예약을 초래한 **정원 변경 자체를 취소** → 정원 원래대로 복원

| # | 테스트 시나리오 | 선행 조건 | 입력값 | 예상 결과 | 검증 항목 | ☑ |
|---|---|---|---|---|---|---|
| 3.1 | 영향 예약 1개, **선택 1(이동)** — 유효한 룸 | R101: 정원 4, OPEN / 예약 1개(R101, partySize=3, 2026-03-22 10:00~12:00, RESERVED) / **대상룸 R102: OPEN, 정원 6(≥3), 동일 시간대 예약 없음** | 1) 새 정원: 2, 2) 선택 1(이동), 3) 대상룸: R102 | R101의 예약이 R102로 이동 → R101 정원 2로 변경 완료 | maxCapacity(R101)=2, roomId(예약)=R102, status=RESERVED 유지 | |

| 3.3 | 영향 예약 1개, **선택 0(전체 취소)** | R101: 정원 4, OPEN / 예약 1개(R101, partySize=3, 2026-03-22 10:00~12:00, RESERVED) | 1) 새 정원: 2, 2) 선택 0(전체 취소) | 정원 변경 자체가 취소됨 → R101 정원 4 원상복원, 예약 유지 | maxCapacity(R101)=4 유지, 예약 RESERVED 유지 | |
| 3.4 | 영향 예약 2개, 각각 **선택 1(이동)** 처리 | R101: 정원 4, OPEN / 예약 2개(R101, partySize=2, 2026-03-22 10:00~12:00, RESERVED) (R101, partySize=3, 2026-03-22 14:00~16:00, RESERVED) / **대상룸 R102: OPEN, 정원 6(≥3), 두 시간대 모두 예약 없음** | 1) 새 정원: 1, 2) 첫 예약: 선택 1 → R102, 3) 두 번째 예약: 선택 1 → R102 | 두 예약 모두 R102로 이동 → R101 정원 1로 변경 완료 | maxCapacity(R101)=1, 두 예약 모두 roomId=R102 반영 | |

## 테스트 실행 방법

1. **테스트 환경 준비**
   ```bash
   ./gradlew clean check  # 회귀 테스트 실행
   D:\ICEP\out> java -jar study-room-cli.jar  # 애플리케이션 실행
   ```

2. **관리자 로그인**
   - userId: `user001`
   - password: `admin1234`

3. **룸 컨디션 관리 메뉴 접근**
   - 관리자 메뉴 → 룸 상태 변경

4. **각 경로별 테스트 실행**
   - Path 1~2: 정원 변경 (영향 예약 없음)
   - Path 3: 정원 변경 (영향 예약 발생) → 예약 이동/취소/롤백
   - Path 4: 임시 휴업 (영향 예약 발생) → 예약 이동/취소/롤백
   - Path 5: 룸 상태 변경 (영향 예약 없음)
   - 각 테스트마다 **검증 항목**의 데이터 파일(`data/rooms.txt`, `data/reservations.txt`) 확인

5. **검증 완료**
   - 모든 ☑ 항목에 체크 표시
   - 예상 결과와 실제 결과 일치 확인
