# 구현작업내용_스터디룸예약CLI

## 1. 문서 목적

본 문서는 2026-03-10 기준으로 진행한 **가상 현재 일시 기반 스터디룸 예약·이용 관리 시스템(Java CLI)** 1차 구현 작업 내용을 정리한 기록 문서다.

## 2. 작업 범위

이번 작업에서는 1차 기획서 기준의 핵심 기능을 실제 실행 가능한 Java CLI 형태로 반영했다.

- member / admin 분리 메뉴 구성
- 회원가입, 로그인, 로그아웃
- 공용 가상 현재 시각 조회 및 변경
- 일반 텍스트 파일 4종 읽기, 검증, 저장
- 스터디룸 목록 조회
- 조건 기반 사용 가능 룸 검색
- 예약 생성
- 내 예약 조회
- 예약 취소
- 체크인
- 30분 1회 사용 연장
- 패널티 조회
- 관리자 전체 예약 조회
- 관리자 사용자 패널티 조회 및 초기화
- 관리자 룸 상태 변경
- 자동 `NO_SHOW`, `COMPLETED` 상태 갱신

## 4. 구현 상세

### 4.1 실행 진입점

- `Main.java`에서 CLI 앱을 시작하도록 구성했다.
- 프로그램 시작 시 필수 데이터 파일을 읽고 검증한다.
- 시작 검증 실패 시 파일 오류 메시지를 출력하고 종료한다.

### 4.2 도메인 구성

- 사용자 역할: `member`, `admin`
- 룸 상태: `OPEN`, `CLOSED`, `MAINTENANCE`
- 예약 상태: `RESERVED`, `CANCELLED`, `CHECKED_IN`, `COMPLETED`, `NO_SHOW`
- 사용자, 룸, 예약, 전체 데이터셋을 각각 별도 객체로 분리했다.

### 4.3 저장 구조 구현

다음 4개 파일을 기준 저장소로 사용한다.

- `users.txt`
- `rooms.txt`
- `reservations.txt`
- `system_time.txt`

반영한 저장 정책은 다음과 같다.

- `|` 구분자 사용
- 텍스트 필드 이스케이프 처리 지원
- 전체 파일 로딩 후 문법 규칙 검증
- 전체 파일 로딩 후 의미 규칙 검증
- 저장 시 임시 파일 작성 후 재검증
- 재검증 성공 후 원본 교체
- 검증 실패 시 원본 유지

### 4.4 가상 현재 시각 정책 반영

- 실제 시스템 시간은 사용하지 않는다.
- 현재 시각 기준은 `system_time.txt`의 `NOW|yyyy-MM-ddTHH:mm` 한 줄이다.
- 현재 시각 변경은 로그인 후 메뉴에서만 가능하다.
- 새 현재 시각은 기존 값보다 과거일 수 없다.
- 현재 시각 변경 직후 자동 상태 갱신을 수행한다.

### 4.5 자동 상태 갱신 규칙 반영

다음 상황에서 자동 상태 갱신이 실행되도록 구성했다.

- 프로그램 시작 직후
- 로그인 직후
- 현재 시각 변경 직후
- 예약/패널티/관리자 예약 조회 관련 기능 진입 전
- 관리자 패널티 초기화 전
- 관리자 룸 상태 변경 전

적용 규칙은 다음과 같다.

- `RESERVED` 예약에서 시작 후 15분이 지나면 `NO_SHOW`
- `NO_SHOW` 전환 시 패널티 1점 증가
- `CHECKED_IN` 예약에서 종료 시각 이상이 되면 `COMPLETED`

## 5. 기능별 반영 내용

### 5.1 계정 기능

- 회원가입은 `member`만 생성한다.
- admin은 데이터 파일에 사전 등록된 계정만 사용한다.
- 로그인 성공 후 역할별 메인 메뉴로 분기한다.

### 5.2 예약 기능

- 예약 시작/종료 시각은 30분 단위만 허용한다.
- 초기 예약 길이는 1시간 이상 4시간 이하만 허용한다.
- 현재 시각 기준 14일 이내의 미래 예약만 허용한다.
- 같은 룸 시간대 중복 예약을 금지한다.
- 같은 사용자 시간대 중복 예약을 금지한다.
- member의 미래 예약은 최대 2개로 제한한다.
- 패널티가 2점 이상이면 새 예약 생성을 막는다.

### 5.3 체크인 및 연장 기능

- 체크인은 예약 시작 10분 전부터 시작 후 15분까지 허용한다.
- 체크인 성공 시 `CHECKED_IN`으로 전환하고 체크인 시각을 저장한다.
- 연장은 `CHECKED_IN` 상태에서만 가능하다.
- 연장은 30분 1회만 허용한다.
- 뒤 예약이 있거나 룸 운영 종료 시각을 넘으면 연장을 막는다.

### 5.4 관리자 기능

- 전체 예약 조회
- member 패널티 목록 조회
- member 패널티 0점 초기화
- 룸 상태 변경
- 현재 또는 미래의 활성 예약이 남아 있으면 `CLOSED` 또는 `MAINTENANCE` 전환을 막는다.

## 6. 검증 및 실행 확인

### 6.1 빌드 확인

아래 명령으로 컴파일 성공을 확인했다.

```bash
javac src/*.java
```

### 6.2 실행 시나리오 확인

다음 흐름을 실제 실행으로 확인했다.

1. 회원가입 → 로그인 → 예약 생성
2. 관리자 현재 시각 변경 → 자동 `NO_SHOW` 및 패널티 반영
3. member 체크인 → 30분 연장 → 시간 이동 후 `COMPLETED` 전환
4. 활성 예약이 남아 있는 룸에 대해 `CLOSED` 변경 시도 → 차단 확인
5. 관리자 패널티 조회 및 초기화
