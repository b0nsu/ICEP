# 시간관리 파트 추가 TC 후보

## 추가 TC 후보

| ID | 세부 목표 | 입력값 | 결과 |
|---|---|---|---|
| TC38 | 현재 가상 시각 입력 형식 오류 처리 | member 로그인 상태에서 현재 시각 변경 시 새 현재 시각으로 `2026/03/20 09:00` 입력 | `오류: 날짜/시각 형식이 올바르지 않습니다. 예: 2026-03-20 09:00`를 출력하고 member 메뉴로 돌아간다. |
| TC39 | 존재하지 않는 일시 입력 거절 | 현재 가상 시각 `2026-03-20 09:00` 상태에서 새 현재 시각으로 `2026-02-30 09:00` 입력 | `오류: 날짜/시각 형식이 올바르지 않습니다. 예: 2026-03-20 09:00`를 출력하고 member 메뉴로 돌아간다. |
| TC40 | 현재 가상 시각 입력값 앞뒤 공백 거절 | member 로그인 상태에서 현재 시각 변경 시 새 현재 시각으로 ` 2026-03-20 10:00` 또는 `2026-03-20 10:00 ` 입력 | `오류: 입력값 앞뒤에 공백을 넣을 수 없습니다.`를 출력하고 member 메뉴로 돌아간다. `system_time.txt`는 변경되지 않는다. |
| TC41 | 예약 시작 시각이 현재 가상 시각과 같은 경우 거절 | 현재 가상 시각 `2026-03-20 09:00`, 날짜 `2026-03-20`, 시작 `09:00`, 종료 `10:00`, 인원 `2`, 룸 `R101` 입력 | `오류: 예약 시작 시각은 현재 가상 시각보다 미래여야 합니다.`를 출력하고 member 메뉴로 돌아간다. |
| TC42 | 체크인 가능 구간 하한 경계 성공 | 현재 가상 시각 `10:50`, 예약 시작 `11:00`, 상태 `RESERVED`, 룸 상태 `OPEN`인 예약번호 `rv0001`로 체크인 시도 | `체크인이 완료되었습니다.`를 출력하고 예약 레코드가 `CHECKED_IN`으로 반영된다. |
| TC43 | 체크인 예약번호 형식 오류 처리 | 체크인 예약번호 입력값으로 `RV0001` 또는 `rv001` 입력 | `오류: 예약번호 형식이 올바르지 않습니다. 예: rv0001`를 출력하고 member 메뉴로 돌아간다. |

## 선택 반영 후보

| 후보 | 검토 메모 |
|---|---|
| 체크인 실행 직전 자동 상태 갱신 후 NO_SHOW 예약 체크인 거절 | `11:11` 이후 체크인을 시도하면 기능 수행 직전 자동 상태 갱신으로 예약이 `NO_SHOW`가 된 뒤 체크인 가능한 상태가 아니라고 거절되는 흐름이다. 다만 기존 자동 상태 전환 TC와 일부 연결되므로 선택 반영으로 둔다. |
| 예약 시작=종료, 종료<시작 | 의미 있는 시간 구간 오류이지만 기존 예약 시간 조건 TC와 묶어 검토하는 것이 더 자연스러워 이번 우선 후보에서는 제외했다. |
| 예약 날짜/시각 형식 오류 | 입력 형식 검증 보강용으로는 가능하지만, 보고서 분량을 고려하면 현재 시각 형식 오류와 체크인 예약번호 형식 오류를 우선하는 것이 낫다고 판단했다. |
