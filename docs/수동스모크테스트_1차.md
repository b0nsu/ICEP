# 수동 스모크 테스트 (1차)

기준 빌드: `./gradlew clean check`

## 실행 확인
- 명령: `printf '0\n' | ./gradlew run`
- 결과: 비로그인 메뉴 출력 후 정상 종료

## 핵심 시나리오

1) 잘못된 로그인
- 입력: `memb1 / Wrong1!`
- 결과: `[실패] 로그인 실패: 비밀번호가 일치하지 않습니다.`

2) 정상 로그인
- 입력: `memb1 / Pass123!`
- 결과: `[성공] 로그인되었습니다. 역할: MEMBER`

3) 예약 신청 성공
- 입력: `memb2`, `R102`, `2026-03-21`, `15:00-16:00`, `2`
- 결과: `[성공] 예약 신청 완료: RV0004`

4) 시간 충돌 예약 거부
- 입력: `memb1`, `R101`, `2026-03-20`, `11:30-12:30`
- 결과: `[실패] 같은 room/date의 예약 시간이 충돌합니다.`

5) 잘못된 roomId 거부
- 입력: `R999`
- 결과: `[실패] 존재하지 않는 roomId입니다.`

6) 체크인 허용 구간 밖 거부
- 현재 시각 `2026-03-20 10:00`, 대상 예약 시작 `11:00`
- 결과: `[실패] 체크인 허용 구간이 아닙니다.`

7) 가상 현재 시각 변경 후 자동 전이
- ADMIN이 `2026-03-20 11:20`으로 변경
- 결과: `RESERVED->NO_SHOW 1건`

8) 룸 capacity 축소 후 ADJUST_REQUIRED 전이
- ADMIN이 `R101` capacity를 `1`로 변경
- 결과: 영향 예약 `RV0001`, 전이 `RESERVED->ADJUST_REQUIRED 1건`

9) ADMIN 예약 조정 성공/실패
- 성공: `RV0003 -> R102`
- 실패: `RV0002 -> A203(CLOSED)`
