# MANUAL TEST SCENARIOS

아래 시나리오는 `./build.sh` 후 `./run.sh` 기준 수동 점검 절차입니다.

## 공통 준비
1. 프로그램 시작
2. 필요 시 `data/*.txt`를 메모장으로 수정
3. 시나리오 별 계정/시간/예약 데이터를 맞춰 진행

## 1) 회원가입 성공
- 비로그인 메뉴 `1. 회원가입`
- 유효한 userId/password/displayName 입력
- 기대 결과: "회원가입이 완료되었습니다." 출력, `data/users.txt`에 추가

## 2) 중복 userId 회원가입 실패
- 이미 존재하는 userId로 회원가입 시도
- 기대 결과: "이미 존재하는 사용자 ID" 오류

## 3) 로그인 성공/실패
- 실패: 잘못된 비밀번호 입력
- 성공: 올바른 userId/password 입력
- 기대 결과: 실패/성공 메시지가 각각 정확히 출력

## 4) 예약 가능 룸 조회 성공
- member 로그인
- 유효한 날짜/시각/인원 입력
- 기대 결과: OPEN + 수용인원 충족 + 비겹침 룸 목록 출력

## 5) 룸 겹침으로 예약 실패
- 같은 룸 같은 시간대(겹침)로 예약 생성 시도
- 기대 결과: "해당 시간대에 이미 예약된 룸" 오류

## 6) 같은 사용자 시간 겹침으로 예약 실패
- 같은 사용자로 다른 룸에 같은 시간대 예약 시도
- 기대 결과: "같은 시간대에 이미 다른 예약" 오류

## 7) 체크인 너무 이른 시점 실패
- 예약 시작 10분보다 이른 현재 시각에서 체크인 시도
- 기대 결과: "아직 체크인 가능한 시간이 아닙니다." 오류

## 8) 체크인 허용 구간 성공
- 현재 시각을 `[시작-10분, 시작+15분]` 범위로 맞추고 체크인
- 기대 결과: 체크인 성공, 상태 `CHECKED_IN`, checkedInAt 저장

## 9) 시간 변경으로 NO_SHOW 전환
- RESERVED 예약(미체크인) 준비
- admin으로 현재 시각을 `시작+15분` 초과로 변경
- 기대 결과: 해당 예약이 `NO_SHOW`로 전환

## 10) 시간 변경으로 COMPLETED 전환
- CHECKED_IN 예약 준비
- admin으로 현재 시각을 종료 시각 이상으로 변경
- 기대 결과: 해당 예약이 `COMPLETED`로 전환

## 11) member 예약 취소 성공
- 본인 RESERVED 예약, 시작 전 상태에서 취소
- 기대 결과: `CANCELLED` 전환

## 12) 시작 후 예약 취소 실패
- 현재 시각을 시작 시각 이상으로 만든 뒤 취소 시도
- 기대 결과: "이미 진행 중이거나 종료된 예약은 취소할 수 없습니다." 오류

## 13) admin 예약 조정 성공
- 미래 RESERVED 예약 선택
- 조건 충족 OPEN 룸으로 이동
- 기대 결과: roomId만 변경, 나머지 정보 유지

## 14) admin 예약 조정 실패(수용 인원 부족)
- partySize보다 정원이 작은 룸으로 이동 시도
- 기대 결과: "대상 룸의 수용 인원이 부족합니다." 오류

## 15) admin 룸 휴업 시 영향 예약 처리
- 미래 RESERVED 예약이 있는 룸을 휴업 시도
- 영향 예약 목록 확인 후 이동/취소 선택
- 기대 결과: 전부 처리 후에만 CLOSED 반영, 중간 0 선택 시 전체 롤백

## 16) admin 룸 수용 인원 감소 시 영향 예약 처리
- 감소 후 초과되는 미래 RESERVED 예약 존재 상태에서 정원 변경 시도
- 영향 예약별 이동/취소 처리
- 기대 결과: 전부 처리 후 정원 반영, 중간 0 선택 시 전체 롤백

## 17) 잘못된 data 파일 line 검출
- 예: `data/users.txt` 한 줄 필드 개수를 의도적으로 깨뜨림
- 프로그램 재시작
- 기대 결과: 파일명+줄번호+원인을 포함한 오류 출력 후 시작 중단

## 18) CLOSED 룸 예약/체크인 거부
- 룸을 CLOSED로 변경
- 해당 룸 예약 생성/체크인 시도
- 기대 결과: 각각 거부 메시지 출력
